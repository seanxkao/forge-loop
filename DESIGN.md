# 工廠迴圈（暫名 Forge Loop）— 雛型設計文件

> 網頁版增量遊戲。工廠優化 ＋ 做裝 ＋ 自動戰鬥。
> 本文件記錄第一個可玩雛型（最小閉環）的設計共識。數值為起始參考值，之後再平衡。

---

## 1. 核心概念

一句話：**打怪掉素材 → 素材餵機台產出製裝材料 → 反覆製裝、骰詞綴讓英雄變強 → 打更難的關（魔王）拿更多素材 → 製裝更快、骰更多。**

```
        ┌─────────────────────────────────────────────┐
        │                                             │
        ▼                                             │
   ┌─────────┐   掉落原始素材   ┌──────────┐  中間材料  ┌─────────┐
   │  戰鬥    │ ───────────────▶│  生產(方格) │ ────────▶│  製裝    │
   │ 波次關卡 │                  │  機台運轉   │           │ 合成裝備 │
   └─────────┘                  └──────────┘           └─────────┘
        ▲                            │                      │
        │                            │ 消耗素材運轉           │ 裝備
        │        裝備強化英雄          ▼                      │
        └──────────────────────── 英雄 ◀────────────────────┘
```

### 設計引擎：魔王牆 ＋ 抽裝成長（best-of-N）

成長**不**靠「換更高階的裝」，而靠在固定裝備上**反覆骰詞綴、留最好的**：

- 每槽只有**一種固定基底**裝備，沒有上位替代；power 全部來自隨機詞綴（數量 1～4 ＋ 數值）。
- 變強 = 骰到更多、更好的詞綴。做得越多次越可能摸到 god roll，所以**生產速度（製裝次數／分鐘）＝成長速度**。
- 素材只能靠打怪掉；後期關卡掉量呈**數量級暴增**，讓玩家瘋狂製裝衝 roll。
- **瓶頸＝每區魔王**：3 關順順爬，第 4 關魔王數值尖峰卡住你，逼你回去骰更好的裝才能突破。

### 進程原則（平衡依據）

- 總時數約 **1～2 小時**，五區**各約 1/5**（每區時間相近）。
- 每區節奏：3 關快速成長 → 魔王牆停滯 → 骰裝突破。
- power 有**天花板**（固定詞綴池、最多 4 詞 × 3 槽，全 god roll 即上限）；敵人難度曲線對齊此上限回推。
- 生產會滾雪球，故**後期魔王相對更厚**（尖峰倍率逐區加碼）以維持「每區時間相近」。

## 2. 核心系統

### 2.1 戰鬥（Combat）

- **單英雄**、**波次關卡**、即時自動戰鬥（玩家不操作單位，戰前決策為主：穿什麼裝、挑哪關）。
- 英雄自動逐波清怪，每隻怪死亡掉落素材；英雄死亡該次中斷。
- **英雄屬性**（皆由裝備推導，裸值極弱）：
  - 生命 `hp`、攻擊 `atk`（點傷加總）、武器本地物理 `localPhysPct`、防禦 `def`、暴擊 `critChance`、暴傷 `critMult`、攻速 `haste`、每秒回血 `hpRegen`、減傷 `dmgReductionPct`、減暴傷承受 `critDmgTakenReductionPct`
- **攻擊公式**（分層，PoE 風）：
  ```
  武器傷害 = (武器基底 + 武器點傷) × (1 + local 物理%)
  總攻擊   = 武器傷害 + 全域點傷（飾品等）
           ×（1 + global 物理%）       ← 未來擴充，現無此詞綴，公式預留插槽
  暴擊時   × 暴傷（critMult）
  命中傷害 = max(1, 總攻擊 − 敵防禦)
  ```
  > `localPhysPct` 為武器**本地**屬性，只乘武器基底＋武器點傷，不可與飾品點傷一起無腦加總；屬性推導需單獨處理武器這件。
- **受擊公式**（敵人攻擊英雄；敵人可暴擊，一般敵人不暴、魔王可暴）：
  ```
  base = 敵攻擊 × (1 − 減傷%)
  base = base − 固定防禦
  敵暴擊時：有效暴傷 = 1 + (敵暴傷 − 1) × (1 − 減暴傷承受%)；base × 有效暴傷
  受傷 = max(1, base)
  ```
  > **上限**：百分比減傷最多 **90%**；暴擊率、local% 等其餘屬性不設上限（待平衡時再視情況加）。
- **每秒回血**：英雄戰鬥中每秒回復 `hpRegen` 點。
- **攻速**：實際攻擊間隔 = 基礎間隔 / (1 + haste)。
- **節奏**：固定 tick（見 §4），各單位用攻擊計時器決定出手。
- **視覺**：Canvas 像素風，英雄左、怪右；攻擊補間、HP 條、跳傷害數字。

### 2.2 生產（Production）

- **無盤面格子**：每種機台一張卡片，卡上控制——「**製造**」（花建造素材，**總台數 +1** 並預設運轉）、「**＋／－**」（配置運轉台數，機台不消失、只在運轉／閒置切換，**不退素材**）。顯示「運轉／總數」X/Y。
- **生產速度 = 基礎速度 × 運轉台數**（閒置機台不耗料、不產出；運轉台數受素材限制即生產速度）。
- 機台運轉：每週期消耗輸入素材、產出中間材料；缺料則閒置（卡片標紅）。
- 3 種機台把 3 種原始素材各自加工成對應中間材料（製裝貨幣）。
- 機台升級（縮週期／增產出）**未實作**（見 §6）；現以「製造更多台」當生產速度軸。
- **視覺**：每張卡顯示機台圖示 ＋ 運轉台數 ＋ 配方 ＋ 進度條 ＋ 製造／－ 鈕。

### 2.3 製裝（Crafting）

- 消耗對應槽的**中間材料**合成裝備（武器吃金屬錠、防具吃皮革、飾品吃精晶；見 §3）。
- **每槽只有一種固定基底**，無階級、無上位替代。基底是小錨，power 全來自詞綴。
- **詞綴 = 隨機數量 ＋ 隨機數值**：
  - 數量每次 roll **1～4 條**，加權偏少、偶爾 4（jackpot）；初版權重 50/30/15/5%（**待平衡**）。
  - 每條詞綴**分 8 階（tier）**，各階有自己的 min/max 與抽中權重（越高階越強、越難中）；抽詞綴時**先依權重抽階、再在該階範圍 roll 值**。從該槽池無重複抽。
    - 初版調性（待平衡）：高階機率**每階約減半**（權重 128→1）；各階範圍**不重疊**（後一階 min 高於前一階 max，整段往上跳）——抽到高階＝明確的數值躍升。敵人曲線已依此重算（§3.2）。
  - **詞綴池固定**（不隨材料變強），單件 power 有上限；成長靠**反覆製裝留最好的**（best-of-N）。
- **製裝機（每槽一台、佇列式）**：製裝**不再瞬發**。武器／防具／飾品各一台製裝機（**起手各 1 台**），比照生產機台——「**製造**」花建造素材**總台數 +1** 並預設運轉、「**＋／－**」配置運轉台數（機台不消失、不退素材）；**製裝速度 = 基礎 × 運轉台數**。
  - 生產分頁中，每個製裝機卡片各自佔一整列，列內直接顯示該槽的**訂單按鈕**與**過濾器**，讓製裝控制集中在同一處。
  - ×1／×10／×100／×1000 按鈕把 N 件加入**該槽製裝佇列**；機台每週期逐件消耗該槽配方材料製裝，**產出走過濾器**（符合進主背包、否則進倉庫）。
  - 佇列空 → 閒置；缺料 → 暫停等料（卡片標紅）。可一鍵**清空佇列**。
  - 建造成本、週期秒數**待平衡**（初版：每台週期 1s、建造 10 中間材料）。
- 裝備穿上即時生效。做壞的可**拆除**，退還該配方 **25% 素材**（小數部分以機率進位，例如 0.75 → 75% 退 1）。
  > 未來：拆除特定詞綴換取高級材料（暫不實作，見 §6）。

### 2.4 研究（Research）

把裝備投入**拆解器**換永久強化（裝備被消耗）。獨立分頁。

- **拆解機**：研究分頁上**可製造、＋／－ 配置運轉台數**的機器（像生產機台）。**拆解速度 = 基礎 × 運轉台數**（每台每週期 2s 取 1 件，待平衡），起手附 1 台。每週期自動從**倉庫**取「含 T3+ 詞綴」的裝備銷毀（**跳過無研究值的爛裝**）；無可拆則閒置。
- **研究值**：銷毀一件時，其上 **T3 以上**的詞綴各依階產生研究值（越高階越多，初版 `(tier−2)×10`）；T1～T2 不產值。
- **研究軌（每類詞綴一條）**：研究值灌進「該類詞綴」的軌（點傷詞綴 → 點傷研究）。累積**不受前置鎖限制**（即使未來研究被前置鎖住，值也先存著、拆裝不浪費），但同一軌一次只推進當前階。
- **分階解鎖**：每階需固定研究值，**達成後下階 ×2**（初版首階 50，待平衡）。
- **加乘**：每完成一階 → 該類詞綴**永久 +10% 強度**；`deriveStats` 時詞綴實際值 = roll 值 ×(1 + 10%×已完成階數)。
- 與手動「拆除」（退 25% 素材）為**不同出口**：拆解器只產研究值、不退素材。
- **基底研究（每槽一條）**：投入研究會**消耗該槽 N 件裝備**（倉庫優先、再主背包；初版首階 5 件、達成後 ×2，待平衡），每完成一階 → 該槽**基底數值永久 +20%**（`deriveStats` 時基底 ×(1＋20%×階數)；與詞綴強度研究獨立）。給爛裝另一個去處。

## 3. 遊戲內容（5 區 × 20 關；3 槽固定裝備）

> 「架構」如下；細部數值（敵人曲線、掉率、詞綴範圍、成本）見 `src/game/content.ts` 與 `affixTable.csv`，標**待平衡**者待實測。
> 核心節奏：3 關快速成長 → 魔王牆 → 骰更好的裝突破 → 下一區。素材**不分階**，後期關卡掉量數量級暴增。

### 3.1 素材（3 種，對應 3 槽；無階級）

| 槽 | 原始素材 | 機台 | 中間材料（製裝貨幣） |
|----|----------|------|----------------------|
| 武器 | 礦石 `ore` | 熔爐 | 金屬錠 `ingot` |
| 防具 | 獸皮 `hide` | 鞣製台 | 皮革 `leather` |
| 飾品 | 晶石 `shard` | 晶化器 | 精晶 `crystal` |

- 三種原始素材**所有關卡都掉**，掉量隨關卡編號上升（後期數量級暴增）。
- 無更高階素材；玩家在工廠分配三條產線。

### 3.2 關卡（20 關 = 5 區 × 4 關，第 4 關為魔王）

| 區（關卡） | 區域 | 一般敵人 | 魔王（第 4 關） |
|------------|------|----------|------------------|
| 1（1–4） | 礦坑帶 | 史萊姆 🟢 | 史萊姆王 |
| 2（5–8） | 荒野帶 | 野狼 🐺 | 狼王 |
| 3（9–12） | 晶洞帶 | 晶化魔像 🗿 | 魔像核心 |
| 4（13–16） | 鐵域帶 | 鋼鐵魔偶 🤖 | 鋼鐵霸主 |
| 5（17–20） | 虛空帶 | 虛空獸 👾 | 虛空之王 |

- 每區前 3 關難度漸增、可順順清；**第 4 關魔王＝瓶頸牆**：魔王 HP/ATK **直接指定**（與一般曲線解耦，便於獨立調；初版第 1 區約 10 下可擊殺，**待平衡**），且**擁有暴擊**（一般敵人不暴）。
- **後期魔王相對更厚**以抵銷生產雪球（尖峰倍率逐區加碼，**待平衡**）。
- 一般關：每關 5 波、每波 1 隻。魔王關：前 4 波各 1 隻雜兵暖身、第 5 波單一魔王。
- **掉落**：三種素材皆掉，量隨關卡上升；魔王**保證掉大量素材**當破牆獎勵。

### 3.3 機台（3 種，各 1 進 1 出；數值待平衡）

| 機台 | 建造成本 | 輸入 → 輸出（每週期） | 週期 |
|------|----------|----------------------|------|
| 熔爐 | 5 礦石 | 2 礦石 → 1 金屬錠 | 3s |
| 鞣製台 | 5 獸皮 | 2 獸皮 → 1 皮革 | 3s |
| 晶化器 | 5 晶石 | 2 晶石 → 1 精晶 | 3s |

### 3.4 裝備（3 槽各一固定基底；詞綴 1～4 隨機）

| 槽 | 名稱（暫名） | 固定基底 | 製作成本 |
|----|--------------|----------|----------|
| 武器 | 劍 | atk 10 | 3 金屬錠 |
| 防具 | 甲 | hp 40 / def 3 | 4 皮革 |
| 飾品 | 護符 | critChance 5% | 2 精晶 |

> 基底與成本為**待平衡**初版。基底刻意小，power 主要來自詞綴。

**詞綴池（固定，PoE 風；每槽 5 種）**：

- **武器**：點傷（flat atk）、local 物理%、暴擊、暴傷、攻速
- **防具**：固定生命、固定防禦、每秒回血、百分比減傷、減少承受暴擊傷害
- **飾品**：點傷、暴擊、暴傷、生命、攻速

- 每次製作從該槽 5 種無重複抽 1～4 條（數量加權，見 §2.3）。
- **量產流**：反覆製作、保留各槽最佳、其餘**拆除**退 25% 素材（避免庫存爆量）。
- **背包 / 倉庫分頁**：裝備頁只保留道具檢視，並在上方以子分頁切換**主背包**與**倉庫**；兩者不再同時顯示。逐件按鈕或**右鍵一鍵**互轉。
- **過濾器**：每槽可設任意多列「必備詞綴 ≥ 指定階」條件（AND）；過濾器放在對應製裝機旁，製作時不符者**自動進倉庫**，另有「套用到現有背包」一鍵把現有不符的掃入倉庫。

**詞綴池資料化**：單一事實來源 `src/game/affixTable.csv`（每槽 × 每詞綴 × **每階**一列，欄位 `slot,stat,label,pct,tier,weight,min,max`）。`affixTable.ts` 以 `?raw` 載入解析、匯出 `affixPool(slot)`；要整批重設改 `scripts/gen-affixes.mjs` 規則區（每條詞綴的第 1 階範圍與每階遞增、各階權重）後 `npm run gen:affixes`。

### 3.5 機台與生產速度

- **無盤面**：每種機台一張卡片；製造增加總台數、＋／－ 配置運轉台數（顯示 X/Y），閒置機台不耗料。
- **起手免費附三種機台各一台**（熔爐／鞣製台／晶化器），一開局即有產出。
- **生產速度＝製造更多台**（受素材限制）。機台升級為未來真方案（見 §6）。

## 4. 技術架構

- **技術選型**：Vite ＋ TypeScript。介面用 DOM，戰鬥畫面用 Canvas 2D。
- **遊戲迴圈**：`requestAnimationFrame` ＋ 固定步長累加器（fixed timestep accumulator），確保戰鬥/生產 tick 速率穩定、與畫面更新解耦。
- **狀態**：單一 `GameState` 物件（plain data），系統函式對其操作；不引入 ECS。
- **模組切分（初版）**：
  - `combat/` — 波次推進、傷害結算、掉落
  - `production/` — 機台 tick、素材消耗/產出
  - `crafting/` — 配方、詞條 roll
  - `inventory/` — 素材與材料庫存
  - `hero/` — 屬性推導、裝備穿脫
  - `ui/` — DOM 介面（生產盤面、庫存、製裝、關卡選擇）
  - `render/` — Canvas 戰鬥繪製（像素 sprite、HP 條、傷害數字）
  - `save/` — localStorage 存讀檔（定期存檔）
- **存檔**：localStorage，定期序列化 `GameState`。載入時以**現行 schema 深層合併**舊檔遷移（缺欄補預設、動態 record 沿用、型別衝突回退預設），**改版不清檔**；僅解析失敗／損毀才重置。
- **離線收益**：**雛型不做**。關卡僅在「開著頁面時」即時推進。離線結算之後再加。

## 5. 雛型範圍（Definition of Done）

雛型完成 = 玩家能完整跑一輪閉環且有感：

- [ ] 選關 → 即時自動戰鬥（像素演出）→ 怪物掉落三種素材進庫存
- [ ] 在 3×3 方格建造 3 種機台、把素材煉成製裝材料
- [ ] 反覆製裝（每槽固定基底 ＋ 1～4 隨機詞綴），保留最佳、丟棄其餘
- [ ] 裝備即時改變戰鬥數值（含新公式：local%、減傷、回血、敵人暴擊）
- [ ] 每區 3 關順爬 → 魔王牆 → 骰更好的裝突破 → 下一區
- [ ] localStorage 存讀檔
- [ ] 體感到「魔王卡住 → 拚生產衝 roll → 突破」的節奏

**不在雛型範圍**：離線/掛機收益、盤面擴建＋機台升級（生產速度真方案）、小隊戰鬥、音效、多英雄、成就/轉生等後期系統。

## 6. 待後續決定 / 未實作

- **機台升級**：縮短週期／提高單次產出（目前只能靠製造更多台）。
- **數值平衡**：敵人曲線、魔王尖峰倍率、詞綴範圍、掉落量、製裝成本、詞綴數量權重均為初版，待實測（重點：god 裝能否打穿第 5 區、每區時間是否相近）。
- **global 物理%** 詞綴：攻擊公式已留插槽，詞綴池暫未加入。
- **拆除詞綴換高級材料**：拆除裝備目前只退 25% 素材；未來改成可拆特定詞綴產出高級料。
- **研究擴充**：拆解器速度升級、含前置的研究樹（目前僅「詞綴強度」軌、無前置）。
- 離線/掛機收益結算（時間戳）。
- 美術資產：英雄真正 sprite、敵人像素圖、待機動畫（目前英雄為方塊拼圖、敵人用圖示）。

### 裝備檢視與比較（2026-05-30）

- 滑鼠移上任一裝備（已裝備、主背包、倉庫）時，會顯示浮動 tooltip。
- Tooltip 會拿該裝備與「當前同槽已裝備物」做比較；若該槽未裝備，則只顯示自身數值。
- 比較以「實際生效後的數值」為準，需計入基底研究與詞綴研究加成，不直接照原始 roll 顯示。
- 武器摘要第一行固定顯示「物理傷害」，公式為 `(武器基底攻擊 + 武器點傷) × (1 + 本地物理%)`，並套用研究加成後再顯示。
- Tooltip 比較列需標出正負差值，讓玩家能直接看出升降，不必手算。
### UX 補充更新（2026-05-30）
- 背包頁只保留「主背包」與「倉庫」切換，以及對應裝備列表；製裝訂單與過濾器不再放在背包頁。
- 製裝機恢復為與材料機接近的橫向卡片排列；每張卡片內同時顯示機台控制、製造控制與佇列。
- 材料機與製裝機都使用相同的批次操作概念：主按鈕顯示目前批次與總花費，右側可切換 `1 / 10 / 100`。
- 製裝配方的基底敘述不直接常駐在卡片上，改為提示文字；滑鼠停留時再顯示實際基底效果。
- 過濾器改為 modal 視窗編輯；開啟時覆蓋主介面並阻擋其他 UI 互動。
- 拆解機只有在「有啟用台數」且「倉庫有裝備」時才會累積進度；否則進度條維持 0。
- 基底研究與詞綴研究共用拆解流程：每拆 1 件裝備，會同時增加該槽位的基底研究進度；若裝備上有 T3 以上詞綴，額外提供對應詞綴研究值。
- 拆解機不再限制只能處理有 T3+ 詞綴的裝備；所有倉庫裝備都可拆，但 T1/T2 詞綴仍不提供詞綴研究值。
- 研究頁中，基底研究顯示在詞綴研究上方，並且同樣具有進度條與升階閃爍提示。
- 主畫面的素材列改為底部固定浮動 bar，遊戲進行時持續可見。
- 戰鬥掉落的素材噴字需做短延遲錯開，避免多筆掉落完全重疊。
- 關卡選擇列表需直接顯示該關會出現的敵人，以及各素材的單次掉落範圍，避免只靠 tooltip 或進場後才得知內容。
- 敵方不再有暴擊機制；魔王改以純數值壓力呈現，不使用額外暴傷倍率。
- 原本的「減暴傷承受」詞綴改為「格檔率」。敵人攻擊命中時，若觸發格檔，則在既有百分比減傷上額外加算 `25%` 減傷後再結算固定防禦。
- 格檔成功時，承傷浮字需加上盾牌符號，並稍微淡化，以便和一般受傷區分。
- 詞綴品質權重曲線改為整數權重：最高階固定 `10`，往低階每層乘上 `1.6` 並四捨五入，讓高階詞綴更常見，同時避免小數權重。
### 輪迴系統補充（2026-05-30）
- 輪迴系統：最後一關最終敵人倒下後跳出通關視窗，玩家可選擇繼續當前局，或直接重開一輪並擇一取得永久加成。
- 輪迴加成共有 3 種：研究成長倍率每層乘上 `0.8`，也就是研究門檻改用 `baseCost * (2 * 0.8^輪迴層數)^研究層數`；最終素材掉落量每層乘上 `1.15`；全能力每層乘上 `1.1`。全能力包含生命、防禦、傷害、攻速、暴擊率、暴傷，並在其他來源結算後再乘算。
- 第二輪開始顯示「輪迴」分頁，顯示目前第幾輪，以及三種輪迴加成的累積層數與當前倍率。
- 原料機、製裝機、拆解機的機台成本採階梯式成長：持有數達到 `10`、`100`、`1000` 後，後續每台價格額外提高 `2.5x`；跨門檻批次購買時逐台累加。

### 核心系統補充（2026-05-31）
- 新增核心系統：所有原料機、製裝機、拆解機與核心機都可鑲嵌 2 顆核心；核心與裝備共用背包、倉庫，但屬於獨立道具類型。
- 新增核心機，用於生產核心；核心機本身也可鑲嵌核心，並吃到核心提供的生產速度、產能、返還材料與工藝類效果。
- 核心固定有 1 條「產能」詞綴，會以既有 8 階權重骰值，數值範圍 `2% ~ 4%`；另外再依 `40% / 40% / 20%` 機率骰出 `0 / 1 / 2` 條變動詞。
- 核心變動詞池包含：生產速度、返還材料、提升詞綴階級、增加詞綴數量，以及六種 tag 權重詞（物理、暴擊、速度、生命、防禦、工藝）；其中提升詞綴階級與權重詞數值範圍皆為 `12.5% ~ 100%`。
- 所有詞條都帶 tag；裝備詞與核心詞會受對應 tag 權重影響。增加某類權重的詞本身同時屬於該分類與工藝分類。素材掉落量與返還材料不屬於六大分類。
- 產能機制比照 Factorio：原料機、製裝機與核心機在主進度推進時同步累積紫色細產能條，滿 `100%` 時額外產出 1 個結果，且額外結果必須獨立重骰；拆解機不累積產能條，而是直接按比例提高該次拆解所得研究值。
- 生產速度只作用於機器；返還材料只作用於有材料輸入的機器；提升詞綴階級、增加詞綴數量與權重只作用於會 roll 物品的製裝機與核心機。提升詞綴階級每次只會對該次產物的其中一個詞綴生效。
- 飾品新增「素材掉落量」詞綴；以上新增的固定詞、變動詞與素材掉落量都需進入研究系統、提示文字、比較顯示、存檔遷移與過濾器。
### 解鎖流程更新（2026-05-31）
- 關卡改為循序解鎖。新存檔初始只顯示第 1 關；每次打贏當前關卡後，才會解鎖並顯示下一關。
- 已解鎖關卡只保留在本輪進度中；輪迴重開後，關卡解鎖進度重置回第 1 關。
- 核心系統改為本輪進度解鎖。打贏狼王後，才會開放核心系統。
- 核心系統未開放前，不顯示各機器的核心插槽，也不顯示核心機與核心研究項目。

### 稀有度系統更新（2026-05-31）
- 一般裝備與核心都新增 `normal / magic / rare` 稀有度欄位；核心只會出 `normal` 或 `magic`。
- 一般裝備的稀有度流程：先以 `50%` 從一般升成魔法；若已是魔法，再以 `25%` 從魔法升成稀有。
- 一般裝備詞數改為綁定稀有度：
  - `normal`：`0` 詞
  - `magic`：`1 / 2` 詞，各 `50%`
  - `rare`：`3 / 4` 詞，機率比 `1 : 2`
- 核心保留固定 `1` 條產能詞；其稀有度只決定額外變動詞數：
  - `normal`：固定詞 + `0` 條變動詞
  - `magic`：固定詞 + `1 / 2` 條變動詞，各 `50%`
- 舊存檔讀取時會依現有詞數補齊稀有度，不清檔：
  - 一般裝備：`0詞=一般`、`1~2詞=魔法`、`3~4詞=稀有`
  - 核心：`1詞=一般`、`2~3詞=魔法`
- 核心詞綴「增加詞綴數量」改為「增加稀有度」，只影響稀有度判定機率。
- 「增加稀有度」詞綴數值範圍改為 `1% ~ 8%`。
- UI 顯示稀有度時以底色標示：一般維持原樣，魔法為藍色，稀有為黃色；套用於已裝備、主背包、倉庫與核心插槽/選單。
### 做裝過濾與核心研究更新（2026-05-31）
- 做裝過濾器新增兩種條件，且與原有條件一律採 AND：
  - 變動詞綴數至少 `1 / 2 / 3 / 4`
  - 稀有度至少 `魔法 / 稀有`
- 「詞數」一律指變動詞綴數；固定詞綴不算在內。
- 核心固定產能詞綴不再列入詞綴過濾器選項，也不參與詞綴研究。
- 固定詞綴視為基底的一部分：
  - 只 roll 數值，不分 tier 強化
  - 吃同類型的基底研究加成
  - 不吃詞綴研究加成
- 新增「核心基底研究」：
  - 拆解核心會累積核心基底研究
  - 核心固定產能詞綴由核心基底研究強化
  - 未來若同類裝備新增其他固定詞綴基底，也共用同一條基底研究

### 防禦穿透（2026-05-31）
- 新增戰鬥屬性「防禦穿透」`defPenPct`，作用是無視對方一部分固定防禦。
- 只影響固定防禦 `def`，不影響減傷率、格檔或其他乘區。
- 實際計算方式：
  - 有效防禦 = `def * (1 - defPenPct)`
  - 傷害再用攻擊力扣除有效防禦
- 玩家預設防禦穿透為 `0%`。
- 敵人預設防禦穿透為 `0%`。
- 只有 `2-4` 的狼王防禦穿透為 `50%`。

- 飾品基底仍維持原本設計；只有飾品「變動詞綴」數值調整。
- 飾品詞綴表的數值區間改為原本的 `1/4`。
  - 因為可裝 `2` 個飾品，整體上限約等於舊版單飾品詞值的 `1/2`。
### 2026-06-01 變更摘要

- 飾品詞綴與其他部位維持獨立資料表，所有飾品詞綴數值整體提升為原本的 1.5 倍。

### 2026-06-01 傳奇核心

- 通關 `2-4` 與 `3-4` 時，各贈送玩家 1 枚傳奇核心。
- 傳奇核心不是一般製造流程的一部分，不能透過核心機或任何正常做裝方式產出。
- 傳奇核心固定帶有 2 條固定詞綴：
  - 一般核心同款的固定產能詞綴
  - `10%` 幸運詞綴階級：當該核心插在機器上時，該機器做出的每條變動詞綴在決定 tier 時，有 10% 機率改為骰兩次並取較高階級。
- 傳奇核心不帶任何其他隨機變動詞綴。
- 傳奇核心不參與研究：
  - 不提供基底研究
  - 不提供詞綴研究
  - 其固定詞綴也不吃研究加成
- UI 以獨立的傳奇底色顯示，和魔法、稀有分開辨識。
